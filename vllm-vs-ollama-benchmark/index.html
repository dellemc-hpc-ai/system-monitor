Key Findings

  • TurboQuant k8v4 throughput bottleneck: Cold start throughput is only 5.2–14.9 tok/s vs fp8's 10.8–23.4 tok/s. The KV cache quantization overhead in TurboQuant causes severe throughput degradation especially for longer outputs.
  • Cold latency advantage of fp8: vLLM fp8 is 2–5× faster in cold latency across all IOLengths. TurboQuant's eager mode enforcement and KV recompression adds significant per-token overhead.
  • Ollama steady-state memory: Uses only 5.5 GB GPU vs ~19 GB for vLLM (both cases) — Ollama is much more memory-efficient but at the cost of throughput.
  • Cache behavior: TurboQuant shows cache slowdown (cold faster than cached) due to KV fragmentation. fp8 and Ollama show expected modest cache speedup.
  • Generation task (200/2000): fp8 cold 12.9s vs TurboQuant 56s vs Ollama 33s — fp8 wins decisively on long-output tasks.
  • Translation (400/400): fp8 2.9s vs TurboQuant 5.3s vs Ollama 3.9s — balanced short-output task.
